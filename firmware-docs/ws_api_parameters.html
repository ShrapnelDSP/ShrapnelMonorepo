Audio parameter JSON messages
Message type Parameters Direction Behaviour Example
initialiseParameters None UI -> Firmware The firmware will respond by sending a parameterUpdate message with the current value of each audio parameter
{
"messageType": "initialiseParameters"
}
parameterUpdate

id (string): The parameter ID of the parameter to change.

value (float): The value of the parameter. This must be in the range 0 - 1.

Any

Firmware: update DSP processing to use the new parameter value.

Frontend: Update UI to show new parameter value.

{
"messageType": "parameterUpdate",
"id": "gain",
"value": 0.5
}
